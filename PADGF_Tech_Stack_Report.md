# PADGF Prototype: Technical Stack Report

This report provides a comprehensive overview of the programming languages, frameworks, libraries, and tools utilized in the development and execution of the **Proactive And Dynamic Gas Fee (PADGF) Prototype** platform.

---

## 1. Core Programming Languages

### **JavaScript (JS)**
JavaScript is the primary backbone of the PADGF prototype, utilized across almost the entire stack.
* **Backend Runtime (Node.js)**: All blockchain simulation logic, script execution, the custom node server, and the core decision engine are written in server-side JavaScript utilizing the CommonJS module system.
* **Frontend Logic (Vanilla JS)**: The dashboard interactions (fetching results, generating UI elements, countdown timers dynamically rendering gas costs) rely entirely on standard browser JavaScript, ensuring lightweight deployment without heavy build systems.

### **HTML5**
* Used for structuring the PADGF Interactive Web Dashboard interface (`index.html`, `user-driven.html`, etc.).
* Facilitates semantic web architecture for the user-facing evaluation panels and data visualization tables.

### **CSS3 (Cascading Style Sheets)**
* Defines the visual styling, dynamic animations, and responsive layout of the dashboard.
* Enhances the experience utilizing modern UI characteristics like glassmorphism (translucency), dynamic hover states, and risk-based color coding (e.g., Green for Low Risk, Orange for Moderate, Red for High).

### **Batch Scripting (.bat)**
* Used for the Windows-based entry-point execution scripts (`Run-PADGF-CLI.bat`, `Run-PADGF-Dashboard.bat`) to automate the startup of the backend node and the frontend server simultaneously.

---

## 2. Core Frameworks & Blockchain Tools

### **Hardhat**
* Provides the local Ethereum environment mimicking mainnet conditions (via mainnet fork feature).
* Handles the compilation and execution of the core simulation scripts (`scripts/user-driven-demo.js`, `scripts/sandwich-attack.js`, etc.).
* **Language context:** Runs on Node.js using JavaScript.

### **Ethers.js**
* A critical JavaScript library used to interact directly with the Ethereum blockchain node.
* It wraps complex cryptographic functions, allows reading of on-chain state, handles token approvals (`ERC20` standard), gas calculations (`getFeeData()`), and executes smart contract transactions directly on decentralized exchanges like Uniswap.

---

## 3. Web Serving Layer

### **Node.js `http` and `fs` Modules**
* The supervisor dashboard is heavily reliant on a lightweight, custom-built Node server (`dashboard/server.js`).
* **`http`**: Manages API routing (e.g., `/api/gas-prices`, `/api/run/user-driven-evaluate`) and serves the initial static webpage.
* **`fs` (File System)**: Handles the persistent extraction and loading of evaluation outputs from `.json` reports generated by the PADGF scripts.
* **`child_process.exec`**: Safely wraps and executes Hardhat terminal commands directly from browser-triggered API calls.

---

## 4. Key Implementation Features by Language

| Module | Primary Language | Purpose |
|--------|-----------------|---------|
| **Risk Evaluator Engine** (`src/riskEvaluator.js`) | JavaScript | Uses mathematical logic to analyze price impact, calculate slippage deviations, and generate normalized risk scores. |
| **DEX Interaction Layer** (`src/dexInteraction.js`) | JavaScript | Utilizes Ethers.js to interface with the blockchain router, parsing output bytes, and handling dynamic gas manipulation logic. |
| **User Dashboard UI** (`dashboard/public/`) | HTML / CSS / JS | Ingests data from backend APIs and handles the dynamic rendering of DOM elements (such as calculating delayed auto-execution flows). |
| **Execution Scripts** (`scripts/`) | Node.js (Hardhat target) | Coordinates the complete start-to-finish workflows (simulating the attacker + victim behavior). |

---

### Summary
The PADGF prototype successfully minimizes its structural footprint by heavily relying on **JavaScript** top-to-bottom. By unifying the backend (Node/Hardhat) and frontend (Vanilla DOM manipulation) under the JS ecosystem, it drastically reduces system requirements while maximizing local execution speed and simulation capability.
